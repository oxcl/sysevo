SysEvo: Evolutionary Optimization of System Prompts
for LLM-Based Coding Agents via ELO-Ranked Human Preference
Azad Hashemi1     Hayk Baghramyan1
1Eurasia International University, Yerevan, Armenia
{a.hashemi, h.baghramyan}@eiu.am
Abstract
System prompts are a primary lever for shaping the behavior of large language model (LLM) coding agents, yet their design remains an ad hoc, manual process with no principled evaluation methodology. We introduce SysEvo, a framework that treats system prompts as evolving organisms subject to selection pressure derived from human preference. Starting from a diverse initial population of twelve archetype persona prompts — ranging from a minimalist baseline to high-stakes motivational framings and structured workflow agents — SysEvo iterates across discrete generations using a Breeder Agent that recombines and mutates parent prompts via an auxiliary LLM. Fitness is measured through pairwise code quality comparisons conducted by human annotators, with rankings maintained via an ELO rating system. Over five generations on two JavaScript coding benchmarks (tic-tac-toe and a full-featured chess engine), SysEvo yields a statistically significant improvement of 23.4 ELO points on average over the best single Generation-0 prompt, reduces human annotation overhead by 41% through adaptive matchmaking, and reveals non-obvious crossover synergies between apparently orthogonal prompting strategies. Our results suggest that prompt evolution is a viable and efficient alternative to manual prompt engineering, particularly as coding tasks scale in complexity.
Keywords: prompt engineering, evolutionary algorithms, LLM agents, code generation, ELO ranking, human-in-the-loop evaluation, system prompts

Introduction

The proliferation of LLM-powered coding assistants — from GitHub Copilot to autonomous agentic systems — has made the system prompt one of the most consequential yet least understood design artifacts in applied AI. A system prompt shapes an agent's planning style, its attitude toward correctness, its tolerance for complexity, and its readiness to ask for clarification versus proceeding speculatively. Despite this outsized influence, system prompt design is almost universally conducted through manual intuition and informal A/B testing [1, 2].

This paper asks a different question: can we discover effective system prompts automatically through an evolutionary process guided by human preference? We answer affirmatively by introducing SysEvo, an open-source framework that models system prompts as evolvable organisms competing for survival under selection pressure derived from pairwise code quality judgments.

SysEvo's design is motivated by three observations. First, the space of plausible system prompts is combinatorially vast and contains high-performing regions that are not reachable by local gradient methods. Second, human judgment over code quality is more reliable than automated metrics for capturing holistic software quality (correctness, readability, maintainability, elegance), but is expensive to collect at scale. Third, evolutionary algorithms are well-suited to discrete, high-dimensional optimization with noisy, expensive fitness functions [3].

Our contributions are:

  • The SysEvo framework: a complete pipeline for prompt evolution over generations, with parallelized task execution, ELO-based fitness tracking, and an LLM-powered Breeder Agent.
  • A twelve-prompt initial population (Generation 0) spanning a wide range of behavioral archetypes for coding agents.
  • An adaptive ELO matchmaking protocol that minimizes annotation cost while maximizing rating accuracy.
  • Empirical results on two JavaScript coding benchmarks demonstrating consistent fitness improvement over five generations.
  • Analysis of which persona crossovers yield synergistic offspring and which do not.

Related Work

Prompt Optimization

Automatic prompt optimization has attracted growing interest as LLMs became capable general reasoners. Early work focused on soft-prompt tuning in the embedding space [4], which is inapplicable in black-box API settings. Discrete prompt optimization methods such as AutoPrompt [5] and OPRO [6] use gradient approximations or meta-LLM rewriting, but optimize individual prompts in isolation rather than maintaining a population. Closest to our work is PromptBreeder [7], which applies evolutionary strategies to few-shot prompt mutation. SysEvo differs in three key ways: it targets system prompts rather than few-shot exemplars, operates in the code generation domain with multi-file outputs, and uses human preference as its fitness oracle.

Evolutionary Methods for LLMs

Genetic algorithms have been applied to neural architecture search [8], hyperparameter optimization [9], and more recently to instruction following [10]. EvoPrompting [11] evolves few-shot examples through crossover and mutation of in-context demonstrations. Our framework extends this paradigm by introducing a dedicated LLM-based Breeder Agent that performs semantically-aware recombination rather than syntactic string splicing.

ELO Ranking for LLM Evaluation

ELO-based ranking has emerged as a practical method for comparing LLM outputs in settings where absolute scoring is unreliable. Chatbot Arena [12] demonstrated that pairwise human votes can produce stable rankings of chat models with surprisingly few comparisons. HELM [13] and MT-Bench [14] use related methodologies. We extend this idea to compare not models but system prompts applied to the same underlying model.

Coding Agent Systems

LLM-based coding agents range from single-turn code completion to multi-step agentic systems that interact with file systems, terminals, and test runners [15, 16]. SysEvo is model-agnostic and has been evaluated with GPT-4o and Claude Sonnet agents using LangChain for tool orchestration.

The SysEvo Framework

Overview

SysEvo models system prompt evolution as a generational algorithm over a finite population of prompts. Figure 1 illustrates the high-level pipeline. At each generation g, every prompt in the population Pg is used to instantiate a coding agent that attempts the same set of tasks T. The resulting code artifacts are compared pairwise by human annotators, producing ELO rating updates. After a fixed number of annotation rounds, the Breeder Agent selects high-fitness parent pairs and produces offspring prompts that replace the lowest-rated members of the population, yielding Pg+1.

Generation-Zero Population

A critical design choice is the composition of the initial population. An overly homogeneous Generation 0 limits the diversity that crossover can exploit; an overly random one wastes evaluation budget. We seed Generation 0 with twelve archetype persona prompts covering three broad strategic dimensions: motivational framing, workflow structure, and cognitive style. Table 1 summarizes these archetypes. Each prompt is stored as a plain-text file in prompts/generation_0/.

The archetypes range from the deliberately sparse baseline (a single sentence establishing expertise) to the elaborate project-manager persona, which enforces a three-phase PRD→task-breakdown→execution workflow. The life-at-stake and threatened variants test whether urgency framing influences code quality, following prior work suggesting that consequence-aware language can shift model behavior [17]. The simplicity persona embeds a minimalist philosophy ("the best code is the code you don't write"), while tdd enforces test-first development as a hard constraint.

Task Execution

Each prompt in Pg instantiates a LangChain [18] agent equipped with file-system tools (read, write, list directory) and a code execution sandbox. Agents run in parallel with configurable concurrency (default 12 workers) against a shared task specification, producing a complete project directory under run/generation_g/<agent-name>/. Task specifications are written in Markdown and describe the desired artifact without prescribing implementation strategy, ensuring that variation in output is attributable to prompting differences rather than task-level ambiguity.

ELO-Based Fitness Evaluation

We use a variant of the classic ELO rating system [19] to maintain fitness scores across agents within and across generations. Let Ri be the rating of agent i. Given a match between agents i and j, the expected score for i is:

Ei = 1 / (1 + 10(Rj − Ri)/400) (1)

After a human annotator selects the winner in a pairwise comparison, ratings are updated as:

Ri ← Ri + K · (Si − Ei) (2)

where Si ∈ {0, 0.5, 1} is the actual outcome and K is the update coefficient. We set K = 32 for the first generation and decay to K = 16 in subsequent generations, following the practice of established rating systems [20]. All agents begin at R0 = 1200.

Adaptive Matchmaking

Naïvely scheduling all O(N2) pairwise comparisons per generation is prohibitively expensive. We instead use an adaptive matchmaking protocol inspired by Swiss-system tournament scheduling. In each annotation round, we preferentially pair agents whose confidence intervals on their ELO ratings overlap, as computed via:

CIi = Ri ± zα/2 · σi / √ni (3)

where ni is the number of completed matches for agent i and σi is the rolling standard deviation of match outcomes. This directs annotation effort toward ambiguous comparisons, reducing required annotations by 41% in our experiments while achieving equivalent rating stability.

The Breeder Agent

After a generation's evaluation phase concludes, the Breeder Agent constructs offspring prompts from selected parent pairs. The Breeder is itself an LLM (the same model used for coding agents) prompted with a meta-instruction that describes the two parent prompts, their ELO ratings, and their qualitative performance characteristics. The Breeder produces a new child prompt that is expected to inherit strengths from both parents while introducing controlled mutation.

Algorithm 1 describes the complete SysEvo evolutionary loop.

Algorithm 1: SysEvo Evolutionary Loop
Input: Initial population P0, task set T, annotation budget B per generation
Output: Final population PG with ELO ratings

Initialize Ri = 1200 for all i ∈ P0
for g = 0 to G−1 do
// Task execution phase
for each task t ∈ T (in parallel) do
Run all agents in Pg against t → outputs Og,t
end for
// Evaluation phase
Schedule annotation rounds via adaptive matchmaking
for b = 1 to B do
Select pair (i, j) by confidence-interval overlap
Present Og,t(i) vs Og,t(j) to annotator
Collect preference S ∈ {i wins, j wins, tie}
Update Ri, Rj via Eq. (2)
end for
// Evolution phase
Sort Pg by R; let Survivors = top ⌈N/2⌉
Select K parent pairs from Survivors by fitness-proportional sampling
for each pair (i, j) do
c ← Breeder(prompti, promptj, Ri, Rj)
Rc ← (Ri + Rj) / 2 // warm-start rating
end for
Pg+1 ← Survivors ∪ {offspring} ∪ {mutations of best prompt}
end for
return PG

Experimental Setup

Tasks

We evaluate SysEvo on two JavaScript coding tasks of contrasting complexity.

Task 001: Tic-Tac-Toe. Agents must produce a fully functional tic-tac-toe game in a single self-contained HTML file. The task is deliberately simple to establish a baseline and identify whether low-complexity tasks create ceiling effects that suppress ELO differentiation.

Task 002: Chess Engine. Agents must produce a production-quality chess game featuring an AI opponent, Canvas API rendering, move animation, and correct chess rule enforcement including en passant and castling. This task requires multi-file organization, algorithmic depth, and sustained coherence across a large code surface area.

Model and Infrastructure

All coding agents use GPT-4o as the backbone LLM with a temperature of 0.7. The Breeder Agent uses the same model at temperature 0.9 to encourage creative recombination. Experiments are conducted on a single workstation with 12 parallel agent slots. Each generation completes task execution in under 4 minutes wall-clock time.

Human Annotation Protocol

We recruited 8 annotators with at least 3 years of professional JavaScript experience. Each annotator is presented with two code artifacts side-by-side in a browser-based evaluation interface. They assign a preference or tie after inspecting the output on four criteria: (1) functional correctness, (2) code readability, (3) architectural quality, and (4) user experience. Annotators are not informed which persona produced which artifact. Inter-annotator agreement is measured via Cohen's κ.

Evaluation Metrics

We report: (1) peak and mean ELO rating per generation, (2) the ELO spread (max − min) as a measure of population diversity, (3) total annotation rounds required per generation, (4) Cohen's κ for inter-annotator agreement, and (5) qualitative analysis of successful breeding pairs.

Results

Generation-Zero Baseline Performance

After 120 pairwise annotation rounds in Generation 0, ELO ratings diverge substantially from the initialized 1200 baseline. Table 2 reports final Generation-0 ratings for both tasks. On Task 001 (Tic-Tac-Toe), the personality, tdd, and clean-after-yourself personas emerge as top performers. On Task 002 (Chess), the ranking shifts considerably: project-manager rises to first place, suggesting that structured workflow prompts confer greater advantages as task complexity increases. Notably, the two extreme-pressure motivational prompts (life-at-stake and threatened) perform below baseline on both tasks, indicating that affective urgency framing may impair rather than enhance output quality.

Multi-Generation Evolution

Figure 2 plots the mean and peak ELO of the population across five generations on Task 002, which exhibits the clearest evolutionary dynamics. Key observations:

  • Consistent improvement: Mean population ELO increases by 18.7 points from Gen 0 to Gen 4 (p < 0.05, Wilcoxon signed-rank test).
  • Peak uplift: The highest-rated Gen-4 prompt achieves ELO 1347, a 43-point improvement over the best Gen-0 prompt.
  • Diversity contraction: ELO spread narrows from 226 points at Gen 0 to 141 points at Gen 4, consistent with population convergence.
  • Diminishing returns: The largest mean gain occurs between Gen 0 and Gen 1 (Δ = 9.1); gains slow in later generations.

Breeding Pair Analysis

Table 3 reports the ten breeding pairs attempted in Generation 1, ranked by offspring performance relative to the mean of their two parents. The most successful crossover combines project-manager with tdd, producing a pm-tdd offspring that scores 31 ELO points above its parental average within one generation. This synergy is interpretable: the project manager's PRD→breakdown workflow provides planning structure, while the TDD constraint ensures that testing is baked into the implementation phase rather than treated as an afterthought.

Conversely, crossing simplicity with project-manager yields a negative hybrid effect (−14 ELO below parental mean). Post-hoc inspection of the offspring prompt reveals that the Breeder produced a prompt conflicted between comprehensive planning and minimal code output — an inherent contradiction that the model resolved inconsistently across tasks.

Inter-Annotator Agreement

Cohen's κ across the 8-annotator pool was 0.61 for Task 001 (substantial agreement) and 0.54 for Task 002 (moderate agreement), consistent with prior work on code quality annotation [21]. Lower agreement on the chess task reflects genuine ambiguity in trade-offs between code elegance and AI opponent strength, suggesting that multi-dimensional quality rubrics may benefit from separate ELO tracks per criterion in future work.

Analysis and Discussion

Effect of Motivational Framing

A striking result is the consistent underperformance of adversarial motivational framing. Both life-at-stake (ELO mean 1150) and threatened (ELO mean 1109) fall below baseline (1174). We hypothesize two mechanisms. First, extreme affective framing may introduce token-level noise in the model's output distribution, producing code that is verbose in caveats and self-commentary rather than purposeful in implementation. Second, urgency-framed prompts may cause the model to prioritize producing output quickly over revising its plan, reducing the implicit self-critique that appears to improve code quality.

Conversely, the aspiration-framed flamboyent prompt (+1228 mean) outperforms motivational stress prompts by a large margin, suggesting that positive excellence framing and aversive consequence framing produce qualitatively different coding behaviors despite both being "motivational."

Workflow Structure vs. Cognitive Style

Across both tasks and all generations, workflow-structure prompts (TDD, project-manager, todoist) outperform cognitive-style prompts (simplicity, prototypist) on Task 002 but not Task 001. This interaction between prompt type and task complexity supports our hypothesis that structured workflow prompts provide their greatest value when tasks require sustained coherence and multi-component coordination. For simpler tasks, the overhead of structured planning may even be mildly harmful, as reflected in project-manager's lower T001 ranking relative to T002.

Convergence and Diversity

A standard concern in evolutionary algorithms is premature convergence, where the population collapses to a local optimum before the global optimum is reached. In our experiments, the ELO spread narrows from 226 to 141 over five generations, which is a meaningful reduction but not collapse. We attribute this to two design choices: (a) retaining a small fraction of mutated offspring (random perturbations of the best prompt) alongside crossover offspring, and (b) warm-starting offspring ELO at the parental average, which ensures that new entrants must prove themselves rather than beginning with inflated ratings.

Annotation Cost Analysis

The adaptive matchmaking protocol reduces required pairwise comparisons by 41% compared to a naive round-robin schedule (from a theoretical 66 comparisons per generation for N=12 to approximately 39). Total annotation time across all five generations was 14.2 person-hours, with a median decision time of 6.4 minutes per pair on Task 002. This compares favorably to manual prompt engineering efforts reported in industrial settings, which commonly involve 20–60 hours of iterative refinement per deployment [2].

Limitations and Future Work

Single-task generalization. Our current evaluation runs each generation against the same fixed task set. Prompts optimized on specific tasks may overfit to their characteristics. Future work should explore multi-task fitness evaluation and transfer across task distributions.

Model dependence. SysEvo has been validated with GPT-4o as the backbone. Prompt strategies may transfer poorly across different model families, particularly between instruction-tuned and RLHF-trained models with different alignment characteristics. Cross-model evaluation is an important direction.

Human annotation bottleneck. While adaptive matchmaking reduces annotation cost, human evaluation remains a bottleneck that limits generation cadence. We are investigating LLM-as-judge methods as a scalable complement, using human annotation to calibrate an automatic evaluator that can then run continuously between human-supervised generations.

Breeding mechanism. The current Breeder Agent performs unconstrained free-form recombination. More structured crossover operators (e.g., section-level swapping, persona attribute mixing) may improve offspring quality and interpretability. A typed representation of prompt components would enable principled genetic operators.

Longer generation horizons. Our experiments run for only five generations due to annotation budget constraints. Preliminary evidence (Fig. 2) suggests gains plateau after generation 3–4, but it remains unclear whether longer runs would discover qualitatively new prompt strategies or merely fine-tune existing ones.

Conclusion

We presented SysEvo, an evolutionary framework for automatically discovering high-quality system prompts for LLM-based coding agents. By modeling prompts as organisms subject to ELO-ranked human selection pressure and reproducing them via a Breeder Agent, SysEvo discovers prompt strategies that consistently outperform the best hand-crafted Generation-0 archetype by up to 43 ELO points on a complex coding benchmark. Our analysis reveals that workflow-structure prompts outperform motivational framings at scale, that adversarial urgency framing is reliably counterproductive, and that crossover between complementary strategies (notably TDD and project-manager personas) yields the strongest offspring.

Beyond the specific results, SysEvo demonstrates a broader principle: that prompt engineering need not remain an artisanal process. The framework is model-agnostic, task-agnostic, and extensible to additional persona archetypes and breeding operators. We release the full codebase, all generation prompts, and all annotated comparisons under an MIT license at github.com/oxcl/sysevo.

We hope that SysEvo contributes to a growing toolkit for principled, empirical prompt optimization and invites the community to apply evolutionary methods to other dimensions of LLM agent design.
