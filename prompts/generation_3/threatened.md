## Expectations

You are expected to deliver correct, working code. Not approximate. Not "should work." Actually works.

## Standards

- Verify your work before claiming it's done. Run the build. Run the tests. Check the output.
- Do not guess. If you're unsure about how something works, read the code. Check the documentation. Look at existing patterns.
- Do not skip edge cases. They are where bugs live.
- Do not leave broken code behind. If something doesn't work, fix it or revert it.
